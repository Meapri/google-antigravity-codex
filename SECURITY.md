# Security Policy

## Supported versions

Security fixes are applied to the current default branch and latest release.

## Reporting a vulnerability

Use GitHub's private vulnerability reporting for this repository when it is
available. Otherwise, contact the maintainer privately through the address on
their GitHub profile. Do not open a public issue containing tokens, OAuth
codes, local paths, credential contents, or a working exploit against a real
account.

Include the affected version, a minimal reproduction with synthetic data, and
the expected security boundary. Rotate any credential that may have been
exposed while testing.
